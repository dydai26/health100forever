# Інструкція з деплою сайту Health Forever на хостинг Hetzner

Цей проект побудований на базі **React + TypeScript + Vite** і є статичним додатком (Single Page Application).

---

## Варіант 1: Hetzner Web Hosting (Apache)
Найпростіший спосіб розміщення через звичайну послугу веб-хостингу (cPanel / FTP).

### Крок 1: Збірка проекту
У терміналі вашого проекту виконайте команду для створення фінального продукту:
```bash
npm run build
```
Після успішного завершення збірки в корені проекту з'явиться папка **`dist`**. Вона містить готові оптимізовані статичні файли сайту та файл конфігурації переадресації `.htaccess` (який автоматично скопіювався з папки `public`).

### Крок 2: Завантаження на хостинг
1. Підключіться до вашого сервера Hetzner за допомогою будь-якого FTP/SFTP-клієнта (наприклад, **FileZilla**), або через веб-файловий менеджер у панелі Hetzner KonsoleH.
2. Скопіюйте **весь вміст папки `dist`** (включаючи файли `index.html`, папки `assets/` та прихований файл `.htaccess`) у кореневу директорію вашого сайту на хостингу (зазвичай це папка **`public_html`** або **`www`**).

## Варіант 2: Hetzner Cloud VPS (Docker / Nginx)
Якщо ви використовуєте віртуальний сервер (Cloud VPS) і хочете запустити додаток у контейнері.

В проекті вже створено файли `Dockerfile` та `nginx.conf`.

### Крок 1: Збірка Docker-образу
На вашому сервері Hetzner або локально виконайте команду:
```bash
docker build -t health-forever-app .
```

### Крок 2: Запуск контейнера
Запустіть додаток, перенаправивши порт 80:
```bash
docker run -d -p 80:80 --name health-forever health-forever-app
```
Сайт буде доступний за IP-адресою вашого сервера Hetzner.
